# 1. 네이버 트렌드 분석기 — 바이브코딩으로 만들기
> 네이버 API + 클로드 코드로 트렌드 분석 웹앱을 단계별로 구현합니다.

---

## 완성 결과물
- **탭1**: 쇼핑 트렌드 — 키워드 검색 → 상위 상품 가격/판매량 분석 + 차트
- **탭2**: 검색 트렌드 — 데이터랩 API로 키워드 검색량 추이 시각화
- **탭3**: 블로그 버즈 분석 — 블로그 리뷰 수집 → AI 감성분석 + 워드클라우드
- **배포**: GitHub Pages (무료)

---

## 사전 준비

### 1-1. 네이버 API 발급받기

> 가이드 링크: https://scribehow.com/viewer/Naver_Developers_API___qDWQUeY0TZ29kgT6dMoHeA

위 가이드를 따라가도 되고, 아래 요약을 참고하세요.

```
1. developers.naver.com 접속 → 네이버 로그인
2. [Application] → [애플리케이션 등록]
3. 애플리케이션 이름: "트렌드분석기" (아무거나 OK)
4. 사용 API 선택:
   ✅ 검색 (쇼핑, 블로그 검색에 사용)
   ✅ 데이터랩 (검색 트렌드에 사용)
5. 환경 추가: WEB 설정
   → 서비스 URL: https://example.com
6. [등록] 클릭
7. Client ID / Client Secret 복사해서 메모장에 저장
```

**주의사항:**
- API 키는 절대 남에게 공유하지 마세요
- 하루 호출 제한: 검색 25,000건, 데이터랩 1,000건
- 충분히 넉넉하니 실습 중 걱정 안 해도 됩니다

### OpenAI API 키 (AI 분석 기능에 사용)
```
1. platform.openai.com → 로그인
2. API Keys → Create new secret key
3. 키 복사 → 메모장에 저장
```

### 클로드 코드 설치
```bash
npm install -g @anthropic-ai/claude-code
```

## 1-2. 설계 및 단계별 구현 계획

### 왜 단계별로 나누는가?

> **바이브코딩의 핵심 원칙**: 한 번에 다 시키면 AI가 혼란스러워합니다.
> 작은 단위로 나눠서 → 동작 확인 → 다음 단계 추가.

| 단계 | 목표 | 핵심 |
|------|------|------|
| 1단계 | 쇼핑 검색 기본 | API 연동 + 테이블 |
| 2단계 | 통계 + 차트 | 데이터 시각화 |
| 3단계 | 검색 트렌드 탭 | 데이터랩 API |
| 4단계 | 블로그 감성분석 탭 | AI 연동 |
| 5단계 | UI 마무리 | 완성도 |
| 6단계 | 배포 | GitHub Pages |

## 1-3. 단계별 구현 (프롬프트 가이드 포함)

### 1단계: 쇼핑 검색 기본 뼈대
> 💬 **왜 이렇게?** 가장 단순한 기능부터 시작. API 연동이 되는지 확인하는 게 최우선.

**클로드 코드에 입력할 프롬프트:**
```
네이버 쇼핑 검색 API를 사용하는 트렌드 분석 웹앱을 만들어줘. 단일 HTML 파일로.

기본 기능:
- 네이버 Client ID / Client Secret 입력란
- 키워드 검색 → 네이버 쇼핑 API 호출
- 상위 20개 상품을 테이블로 표시 (상품명, 최저가, 판매처, 브랜드, 카테고리)
- 다크 테마, 네이버 그린(#03C75A) 포인트 컬러
- 모바일 반응형

CORS 문제 해결:
- corsproxy.io 프록시 사용해서 브라우저에서 직접 API 호출

파일명: index.html
```

**확인 포인트:**
- [x] 키워드 입력 후 검색 버튼 누르면 상품 목록이 나오는가?
- [x] 상품명, 가격이 정상적으로 표시되는가?
- [x] 모바일에서도 레이아웃이 깨지지 않는가?

**수강생이 막히는 포인트:**

| 증상 | 원인 | 해결 |
|------|------|------|
| "Failed to fetch" 에러 | CORS 차단 | `corsproxy.io` 프록시가 코드에 있는지 확인 |
| 401 에러 | API 키 오입력 | 복붙 시 앞뒤 공백 제거, 헤더명 대소문자 확인 |
| 상품명에 HTML 태그 | 네이버가 `<b>` 태그 삽입 | `replace(/<[^>]*>/g, '')` 처리 |

### 2단계: 가격 통계 + 차트 추가
> 💬 **왜 이렇게?** 숫자만 나열하면 인사이트가 안 보임. 통계 카드 + 차트로 한눈에 파악.

```
검색 결과에 다음 기능을 추가해줘.

가격 통계:
- 최저가, 최고가, 평균가, 중간가를 카드 UI로 표시
- 가격 분포 막대 차트 (Chart.js 사용)

추가 기능:
- 가격순 / 이름순 정렬 토글 버튼
- CSV 다운로드 버튼 (엑셀에서 한글 깨지지 않게 BOM 추가)
- 검색 결과 개수 표시
- 상품 링크 클릭 시 새 탭으로 열기
- 가격대별 분포 도넛 차트 (1만원 미만 / 1~3만원 / 3~5만원 / 5만원 이상)
```

**확인 포인트:**
- [x] 통계 카드 4개가 상단에 보이는가?
- [x] 차트가 정상적으로 렌더링되는가?
- [x] CSV 다운로드 후 엑셀에서 한글이 깨지지 않는가?

---

### 3단계: 검색 트렌드 탭 추가 (데이터랩)
> 💬 **왜 이렇게?** 지금 가격뿐 아니라 "이 키워드가 요즘 뜨는 건지"까지 알아야 진짜 트렌드 분석.

```
새 탭 "검색 트렌드"를 추가해줘.

기능:
- 네이버 데이터랩 검색 트렌드 API 사용
  - API 엔드포인트: https://openapi.naver.com/v1/datalab/search
- 키워드 최대 5개까지 입력 (쉼표로 구분)
- 기간 선택: 최근 1개월 / 3개월 / 1년 / 직접 입력
- 검색량 추이 라인 차트 (Chart.js)
  - 키워드별 다른 색상
  - 호버 시 수치 표시
- 기간 내 검색량 평균, 최고점, 최저점 표시
- 키워드 간 상대적 인기도 비교 바 차트

데이터랩 API 요청 형식:
- POST 방식
- body에 startDate, endDate, timeUnit, keywordGroups 포함
- timeUnit: "week" (주간) 또는 "month" (월간)
```

**확인 포인트:**
- [x] 키워드 2~3개 입력 → 라인 차트에 각각 다른 색으로 표시되는가?
- [x] 기간 변경 시 차트가 업데이트되는가?

**수강생이 막히는 포인트:**

| 증상 | 원인 | 해결 |
|------|------|------|
| 데이터랩 API 에러 | POST 방식인데 GET으로 보냄 | fetch 옵션에 `method: 'POST'` 확인 |
| 빈 차트 | keywordGroups 형식 오류 | `[{groupName: "키워드", keywords: ["키워드"]}]` 형식 확인 |
| 날짜 에러 | 날짜 형식 | `yyyy-mm-dd` 형식인지 확인 |

---

### 4단계: 블로그 버즈 분석 + AI 감성분석
> 💬 **왜 이렇게?** 가격+트렌드에 이어 "사람들이 뭐라고 하는지"까지. 여기서 AI가 등장합니다!

```
새 탭 "리뷰 분석"을 추가해줘.

기능:
- 키워드 입력 → 네이버 블로그 검색 API 호출 (최근 10개)
- 블로그 제목, 요약, 작성일, 링크를 카드 형태로 표시
- OpenAI API 키 입력란 추가
- "AI 감성분석" 버튼:
  - 각 블로그 리뷰를 GPT-4o-mini에 보내서 분석
  - 분석 결과: 감성(긍정/부정/중립), 점수(1~5), 핵심이유(20자 이내)
  - 각 카드에 감성 배지 표시 (긍정=초록, 부정=빨강, 중립=회색)
- 전체 감성 비율 도넛 차트 (Chart.js)
- 긍정률 프로그레스 바
- 자주 언급되는 키워드 워드클라우드 (간단한 태그 클라우드)

OpenAI API 호출 시 사용할 시스템 프롬프트:
"아래 블로그 리뷰를 분석해줘. 감성(긍정/부정/중립), 점수(1-5), 핵심이유(20자이내)를 JSON으로 답해.
형식: {\"sentiment\": \"긍정\", \"score\": 4, \"reason\": \"가성비 좋다는 평가\"}"
```

**확인 포인트:**
- [x] 블로그 카드가 정상 표시되는가?
- [x] AI 분석 버튼 → 각 리뷰에 감성 배지가 붙는가?
- [x] 도넛 차트에 긍정/부정/중립 비율이 나오는가?

> 🎉 **WOW 모먼트!** AI가 리뷰를 자동으로 읽고 긍정/부정을 판단하는 순간.

### 5단계: UI 마무리 + 설정 정리
> 💬 **왜 이렇게?** 기능은 완성. 이제 "쓸 만한 도구"로 다듬는 단계. 완성도가 여기서 갈립니다.

```
전체 UI를 정리해줘.

API 키 관리:
- 상단에 ⚙ 버튼 → 모달 팝업으로 API 키 입력 (항상 보이지 않게)
- 네이버 Client ID/Secret + OpenAI API 키를 한 곳에서 관리
- localStorage에 저장 (새로고침해도 유지)
- 키 입력 여부를 상단에 상태 표시 (✅ 연결됨 / ❌ 미설정)

UX 개선:
- 탭 전환 시 부드러운 페이드 애니메이션
- 각 API 호출 시 로딩 스피너
- 에러 메시지를 한국어로 친절하게 표시
- 빈 상태(검색 전)에 안내 메시지
- 탭마다 간단한 사용법 안내 텍스트

디자인:
- 전체적으로 다크 테마 통일
- 네이버 그린(#03C75A) 포인트 컬러
- 푸터: "Powered by 네이버 API + OpenAI | 바이브코딩 워크숍"
```

### 6단계 (선택): GitHub Pages 배포
> 💬 **왜?** 로컬에서만 쓰면 내 컴퓨터에서만 접근 가능. 배포하면 링크 하나로 어디서든 사용.

```bash
# 터미널에서 순서대로 입력
git init
git add index.html
git commit -m "네이버 트렌드 분석기"

# GitHub CLI로 레포 생성 + 푸시
gh repo create 내아이디/naver-trend --public --source=. --push

# 이후 GitHub에서:
# 레포 → Settings → Pages → Source: main branch → Save
# 1~2분 후 https://내아이디.github.io/naver-trend/ 에서 접속 가능
```

## 1-4. 실제 구현 흐름

### 바이브코딩 실습 순서

```
1. 터미널 열기
2. 프로젝트 폴더 만들기
   mkdir naver-trend && cd naver-trend

3. 클로드 코드 실행
   claude

4. 1단계 프롬프트 복사 → 붙여넣기 → Enter
5. 결과 확인 (index.html을 브라우저에서 열기)
6. 잘 되면 → 2단계 프롬프트 입력
7. 안 되면 → 에러 메시지를 클로드에게 알려주기:
   "이런 에러가 나와: [에러 메시지]. 고쳐줘."
8. 반복...
```

### 프롬프트 작성 팁

| 팁 | 설명 | 예시 |
|-----|------|------|
| **한 번에 하나** | 기능 1개씩 추가 | ❌ "탭 3개 다 만들어줘" → ✅ "쇼핑 검색 탭부터 만들어줘" |
| **구체적으로** | 모호하면 AI가 추측함 | ❌ "차트 넣어줘" → ✅ "가격 분포 막대 차트를 Chart.js로 넣어줘" |
| **확인 후 진행** | 동작 확인 없이 다음 단계 가면 위험 | 매 단계 브라우저에서 테스트 |
| **에러는 그대로** | 에러 메시지를 통째로 복사 | "콘솔에 이 에러가 나와: ..." |

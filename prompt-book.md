## Part 1: 네이버 트렌드 분석기 만들기

### 1단계: 기본 뼈대 — 키워드 검색
> 처음부터 다 요구하면 AI가 혼란스러워합니다. 가장 핵심인 쇼핑 검색부터 만듭니다.

```
네이버 쇼핑 API로 상품 가격을 검색하는 웹앱을 만들어줘. 단일 HTML 파일로.

기본 기능:
- 네이버 Client ID/Secret 입력란
- 키워드 검색 → 쇼핑 API 호출
- 상위 20개 상품을 테이블로 표시 (상품명, 최저가, 판매처, 브랜드)
- 다크 테마, 네이버 그린(#03C75A) 포인트 컬러
- 모바일 반응형

CORS 문제 해결:
- corsproxy.io 프록시 사용
```

✅ **확인**: 검색해서 테이블이 나오면 성공!

---

### 2단계: 가격 통계 + CSV 다운로드
> 데이터를 보는 것만으로는 부족합니다. 통계와 내보내기가 있어야 실무에서 씁니다.

```
검색 결과에 기능을 추가해줘.

- 가격 통계: 최저가, 최고가, 평균가, 중간가를 카드로 표시
- 정렬: 가격순, 이름순 토글
- CSV 다운로드 버튼 (엑셀에서 열 수 있게)
- 검색 결과 개수 표시
- 상품 링크 클릭 시 새 탭으로 열기
```

✅ **확인**: CSV 다운로드해서 엑셀에서 열어보기

---

### 3단계: 블로그 리뷰 감성 분석 탭 추가
> 가격만 보면 반쪽짜리. 고객이 뭐라고 하는지(리뷰)까지 파악해야 완전한 분석. 여기서 AI 등장!

```
새 탭 "리뷰 분석"을 추가해줘.

기능:
- 키워드 입력 → 네이버 블로그 검색 API 호출 (최근 10개)
- 블로그 제목, 요약, 링크를 카드로 표시
- OpenAI API 키 입력란 추가
- "AI 분석" 버튼 → 각 리뷰를 GPT에 보내서 감성분석
  - 감성: 긍정/부정/중립
  - 점수: 1~5
  - 핵심 이유: 한 줄
- 전체 긍정률 바 차트 (Chart.js)

OpenAI 프롬프트:
"아래 블로그 리뷰를 분석해줘. 감성(긍정/부정/중립), 점수(1-5), 핵심이유(20자이내)를 JSON으로 답해."
```

✅ **확인**: AI가 리뷰마다 긍정/부정 판단하는 순간 = WOW 모먼트! 🎉

---

### 4단계: 상품명 SEO 최적화 탭 추가
> 시장 파악(가격) + 고객 반응(리뷰) 다음은 실행. "내 상품명을 어떻게 지을까?"

```
새 탭 "상품명 SEO"를 추가해줘.

기능:
- 키워드 입력 → 쇼핑 API로 상위 상품명 20개 수집
- 상위 상품명에서 자주 나오는 단어 자동 추출 (프론트엔드에서)
- "AI 상품명 생성" 버튼 → GPT에 요청:
  - 핵심 키워드 기반으로 최적 상품명 3개 생성
  - 각 상품명의 글자 수 표시
  - 플랫폼별 적합성: 스마트스토어(50자), 쿠팡(100자), 11번가(100자)

OpenAI 프롬프트:
"아래 키워드와 경쟁 상품명을 참고해서 네이버 쇼핑 SEO에 최적화된 상품명 3개를 제안해줘. 
각각 45자 이내, 핵심 키워드를 앞에 배치, JSON으로 답해."
```

✅ **확인**: AI가 상품명 3개를 자동 생성! 🎉

---

### 5단계: 네이버 데이터랩 트렌드 탭 추가
> 지금 잘 팔리는 건 알겠는데, 앞으로도 잘 팔릴까? 트렌드를 봐야 합니다.

```
새 탭 "트렌드 분석"을 추가해줘.

기능:
- 키워드 최대 5개 입력 (쉼표로 구분)
- 기간 선택: 1개월/3개월/6개월/1년
- 네이버 데이터랩 쇼핑인사이트 API 호출
- 시계열 차트로 검색량 트렌드 표시 (Chart.js, 키워드별 색상 다르게)
- AI 분석 버튼: "이 트렌드 데이터를 분석해서 향후 1개월 예측과 추천을 해줘"
```

✅ **확인**: 여러 키워드의 검색량이 차트로 비교되면 성공!

### 6단계: UI 다듬기 + API 설정 정리
> 기능은 다 됨. 이제 쓸 만한 도구로 다듬는 단계.

```
전체 UI를 정리해줘.

- API 키 입력을 상단 ⚙ 버튼 → 모달로 이동 (항상 보이지 않게)
- API 키 localStorage에 저장 (새로고침해도 유지)
- 탭 전환 시 부드러운 애니메이션
- 로딩 스피너 추가
- 에러 메시지를 사용자 친화적으로 (한국어)
- 푸터에 "Powered by 네이버 API + OpenAI" 표시
```

## 💡 프롬프트 작성 팁

### 1. 한 번에 다 시키지 않는다
```
❌ "네이버 API로 가격 비교 + 리뷰 분석 + SEO + 트렌드 + UTM 빌더 + 대시보드를 
    다 포함한 웹앱을 만들어줘"

✅ "네이버 쇼핑 API로 키워드 검색하는 기본 웹앱을 만들어줘"
   → 확인 → "여기에 가격 통계를 추가해줘" → 확인 → ...
```

### 2. 동작 확인 후 확장한다
각 단계에서 **실제로 브라우저에서 열어보고** 동작을 확인한 다음 다음 단계로.

### 3. 에러가 나면 에러 메시지를 그대로 붙여넣는다
```
✅ "이 에러가 나왔어: Failed to fetch... CORS policy... 
    어떻게 수정해?"
```

### 4. AI에게 역할을 준다
```
✅ "너는 네이버 쇼핑 API 전문가야. 이 코드에서..."
```

## ⚠️ 자주 막히는 포인트

| 문제 | 원인 | 해결 |
|------|------|------|
| "Failed to fetch" | CORS 차단 | corsproxy.io 프록시 적용 확인 |
| 네이버 API 401 | ID/Secret 오입력 | 앞뒤 공백 제거, 대소문자 확인 |
| OpenAI 응답 깨짐 | JSON 형식 미지정 | "JSON으로 답해" 프롬프트에 명시 |
| HTML 태그 노출 | API가 `<b>` 태그 반환 | `replace(/<[^>]*>/g, '')` 처리 |
| 데이터랩 API 안됨 | 별도 신청 필요 | 네이버 개발자 센터에서 "데이터랩" 추가 |
| API 키 사라짐 | 새로고침 시 초기화 | localStorage 저장 기능 (6단계) |

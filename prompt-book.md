# 이커머스 바이브코딩 강의용 프롬프트 북

## 1. 네이버 트렌드 분석기 만들기 — 6단계 프롬프트

### 단계 1. 분석 목표 맥락 정리
**프롬프트**
```text
너는 이커머스 강의용 데이터 전략 멘토다. 아래 정보를 표로 요약한 뒤 네이버 데이터랩으로 조사할 분석 목표를 정의하라.
입력 정보
- 브랜드/스토어: {{브랜드}}
- 판매 카테고리: {{카테고리}}
- 최근 판매·트래픽 이슈: {{이슈}}
- 목표 고객 페르소나: {{타깃}}
요구사항
1. 입력을 마크다운 표 1개로 정리한다.
2. 분석 목적을 한 문장으로 선언하고, 성공 판단 지표 3개를 bullet로 제안한다.
3. 확실하지 않은 항목은 꼭 2개의 명확화 질문으로 되묻는다.
모든 답변은 한국어로 작성한다.
```
**왜 이렇게 나눴는지**: 맥락을 먼저 확정해야 뒤 단계에서 나오는 키워드·데이터 요구사항이 일관되게 유지된다.  
**수강생이 막힐 포인트**: 브랜드·타깃 정보가 빈칸일 때 뭘 적어야 할지 모른다.  
**해결법**: 예시 값을 괄호로 힌트로 넣거나, 기본값을 먼저 채워놓고 수정하도록 안내한다.

### 단계 2. 검색 의도별 키워드 후보 생성
**프롬프트**
```text
너는 네이버 검색 키워드 설계자다. 아래는 이전 단계 요약이다.
<<<STEP1_SUMMARY>>>
요구사항
1. 검색 의도 유형(정보/거래/리뷰/로컬)별로 잠재 키워드 3개씩 제안한다.
2. 각 키워드에 예상 계절성(상/중/하)과 구매 여정 단계(TOFU/MOFU/BOFU)를 라벨링한다.
3. 데이터 근거가 약한 항목은 "추정"으로 표시하고, 필요한 경우 추가 정보 요청 문장 1개를 덧붙인다.
결과는 마크다운 표로 제공한다.
```
**왜 이렇게 나눴는지**: 의도를 나눠야 이후 필터링 단계에서 비교 기준이 명확해진다.  
**수강생이 막힐 포인트**: 검색 의도 레이블이 낯설어 분류를 주저한다.  
**해결법**: 프롬프트에 예시(예: “정보=사용법, 거래=구매”)를 주석처럼 덧붙인다.

### 단계 3. 우선순위 필터링 및 점수화
**프롬프트**
```text
너는 데이터 분석 보조 강사다. 아래는 키워드 후보 표이다.
<<<KEYWORD_TABLE>>>
할 일
1. 각 키워드에 대해 검색량 추정치(1-5), 경쟁 강도(1-5), 브랜드 적합도(1-5)를 점수화한다. 근거를 괄호 안에 짧게 메모한다.
2. (검색량 + 적합도) - 경쟁 강도를 계산해 Focus Score를 만들고 내림차순 정렬한다.
3. Focus Score가 4 이상인 키워드만 남기고 나머지는 "관심 보류"로 이유를 남긴다.
출력은 한국어 표와 2줄 요약으로 구성한다.
```
**왜 이렇게 나눴는지**: 수치화 기준을 먼저 만들면 뒤에서 데이터 수집 범위가 줄어든다.  
**수강생이 막힐 포인트**: 점수 기준을 임의로 정해도 되는지 불안해한다.  
**해결법**: “정확한 수치 없으면 상대 비교 기준을 명시하라”는 문장을 프롬프트에 추가해 안심시킨다.

### 단계 4. 데이터 수집 체크리스트 설계
**프롬프트**
```text
너는 네이버 데이터랩 사용법을 안내하는 조교다. 아래 최우선 키워드 목록을 받아 실측을 준비하라.
<<<PRIORITIZED_KEYWORDS>>>
요구사항
1. 최근 12개월, 최근 30일, 최근 7일 기간으로 나누어 어떤 지표(검색량 지수, 성별 비중, 연령대)를 캡처해야 하는지 체크리스트를 만든다.
2. 각 키워드마다 "데이터랩 경로", "선택 필터", "내보내기 형식"을 표로 안내한다.
3. 직접 수집이 힘든 경우 대체 근거(예: 쇼핑인사이트, 자사 로그)를 제안한다.
결과는 체크리스트 + 표 1개로 작성한다.
```
**왜 이렇게 나눴는지**: 데이터 수집을 명문화해야 실습생이 실제 화면에서 무엇을 봐야 하는지 알 수 있다.  
**수강생이 막힐 포인트**: 네이버 데이터랩 메뉴 이름을 모른다.  
**해결법**: 프롬프트 안에 “(예: 데이터랩 > 검색어 트렌드 > 장기)” 식 경로 예시를 넣어준다.

### 단계 5. 인사이트 구조화
**프롬프트**
```text
너는 성과 분석가다. 아래는 수집된 데이터 표이다.
<<<TRENDS_DATA_TABLE>>>
작업
1. 키워드별 주요 패턴(계절성, 급증/급락, 타깃 격차)을 bullet로 요약한다.
2. 인사이트를 "Observation → Interpretation → Action" 3단 구조로 정리한다.
3. 추가 검증이 필요한 항목은 질문 형태로 남긴다.
출력은 한국어이며 표 1개와 bullet 1세트를 포함한다.
```
**왜 이렇게 나눴는지**: OIA 구조를 강제하면 결과가 보고서 형식으로 정돈된다.  
**수강생이 막힐 포인트**: Observation과 Interpretation을 혼동한다.  
**해결법**: 프롬프트에 “Observation=팩트, Interpretation=의미” 라는 정의를 명시해준다.

### 단계 6. 실행 로드맵 및 검증 포인트
**프롬프트**
```text
너는 이커머스 강의용 실습 코치를 맡았다. 아래 인사이트를 바탕으로 실행 계획을 제시하라.
<<<INSIGHT_SUMMARY>>>
요구사항
1. 빠른 실험(1주), 중기 실행(1달), 장기 전략(1분기)로 나눈 실행 로드맵을 표로 작성한다.
2. 각 액션마다 측정할 지표, 기대 효과, 필요 리소스를 적는다.
3. 실행 전에 확인할 리스크와 검증 방법을 3줄 이내로 덧붙인다.
4. 마지막에 "이번 분석으로 답하지 못한 질문"을 2개 적어 후속 분석거리를 만든다.
```
**왜 이렇게 나눴는지**: 실행 계획을 Horizon별로 나눠야 학습자들이 실전 적용 순서를 이해한다.  
**수강생이 막힐 포인트**: 리소스 추정을 막연해한다.  
**해결법**: “리소스는 인력/예산/툴로 나눠 추정” 같은 힌트를 프롬프트에 추가한다.


## 2. UTM 빌더 만들기 — 4단계 프롬프트

### 단계 1. 캠페인 브리핑 정리
**프롬프트**
```text
너는 퍼포먼스 마케터 브리핑 봇이다. 캠페인 요약 정보를 인터뷰하듯 정리하라.
입력 필드
- 캠페인 이름: {{campaign_name}}
- 목표 채널(예: Meta Ads, 네이버 GFA 등): {{channel}}
- 목표 행동(구매/장바구니/구독 등): {{goal}}
- 핵심 오퍼/혜택: {{offer}}
- 대상 지역 및 언어: {{locale}}
요구사항
1. 누락된 정보가 있으면 우선순위 3개 질문으로 파악한다.
2. 채널별 기본 utm_source/utm_medium 가이드를 제안한다.
3. 출력은 마크다운 표 + 요약 문단 1개다.
```
**왜 이렇게 나눴는지**: 브리핑을 먼저 고정해야 나중에 파라미터 규칙을 재사용할 수 있다.  
**수강생이 막힐 포인트**: 어떤 채널명을 적어야 하는지 헷갈린다.  
**해결법**: 대표 채널 예시를 괄호로 제시하거나 드롭다운식으로 나열한다.

### 단계 2. 파라미터 규칙 정의
**프롬프트**
```text
너는 캠페인 네이밍 룰을 설계하는 조교다. 아래 브리핑을 기반으로 UTM 규칙을 정의하라.
<<<CAMPAIGN_BRIEF>>>
작업
1. utm_source, utm_medium, utm_campaign, utm_content, utm_term 각각에 대해 구성 규칙을 제시한다. (예: 접두사, 구분자, 허용 문자)
2. 채널별 예시 값 최소 2개를 든다.
3. 잘못된 예시 1개와 수정 버전을 보여준다.
4. 텍스트는 한국어 표 1개와 규칙 bullet 1세트로 정리한다.
```
**왜 이렇게 나눴는지**: 규칙을 명문화하면 여러 운영자가 동일한 패턴으로 링크를 만든다.  
**수강생이 막힐 포인트**: 규칙과 실제 값의 차이를 헷갈린다.  
**해결법**: “규칙=템플릿, 예시=실제 값”이라는 한 줄 설명을 넣고, 잘못된 예시를 반드시 포함시킨다.

### 단계 3. 최종 UTM URL 생성
**프롬프트**
```text
너는 자동화된 UTM 빌더다. 아래 규칙과 입력 URL을 사용해 최종 추적 링크를 생성하라.
<<<UTM_RULES>>>
기본 랜딩 URL: {{landing_url}}
추가 파라미터 요구사항: {{additional_params}}
작업
1. 규칙에 맞게 각 utm_* 값을 조합하고 URL 인코딩을 적용한다.
2. 최종 URL, 미리보기(파라미터 표), 복사용 JSON 객체를 동시에 제공한다.
3. 누락 필드가 있으면 이유와 함께 placeholder("TBD")로 표시한다.
```
**왜 이렇게 나눴는지**: 규칙과 입력값을 분리해 두면 자동화 스크립트로도 재사용이 쉽다.  
**수강생이 막힐 포인트**: URL 인코딩이 낯설어 특수문자를 그대로 두려 한다.  
**해결법**: 프롬프트에 “공백은 %20, 한글은 UTF-8 인코딩” 등 간단한 지침을 명시한다.

### 단계 4. QA 및 추적 검증
**프롬프트**
```text
너는 캠페인 QA 담당자다. 아래는 방금 생성한 URL이다.
<<<GENERATED_UTM_URL>>>
작업
1. 쿼리 파라미터를 파싱해 규칙 위반이 있는지 체크리스트로 표시한다.

**왜 이렇게 나눴는지**: 마지막 QA 단계를 강제해야 보고서에 잘못된 소스/미디엄이 쌓이지 않는다.  
**수강생이 막힐 포인트**: 파라미터 파싱을 수동으로 해야 해서 귀찮아한다.  
**해결법**: 프롬프트에 “표로 파싱 결과를 보여달라”라고 명시해 시각적 피드백을 받도록 한다.


## 3. 프롬프트 작성 팁

- **한 번에 다 시키지 않기**: 복잡한 작업을 한 프롬프트에 몰아넣으면 모델이 우선순위를 임의로 정한다. 반드시 맥락→아이디어→필터→실행 순으로 쪼개고, 앞 단계 산출물을 다음 단계 입력으로 넘겨라.
- **확인 후 확장하기**: 각 단계 출력이 기대와 다를 때 즉시 수정 지시를 내리고, “재확인 질문 2개만 해라” 같은 조건을 통해 상호 작용형 워크플로를 유지해라. 출력이 만족스러울 때만 다음 단계로 확장한다.
```

1. 확인 후 파일 저장: `prompt-book.md`.
2. 실습 전 단계별 샘플 입력을 준비하면 강의 진행이 더 수월하다.
